# Functions & Methods

## Key Principles

- Single responsibility - one function, one job
- No global keyword - never manipulate global state
- Type annotations - explicit and readable
- Return values - don't just print
- Raise exceptions - let the caller handle errors
- Raising errors affects reusability - how do we even write reusable functions and classes?
